# Практика 1-6 — финальная структура

Проект разделён на две части:

- `практика 1-6/backend` — Node/Express API. Запуск:
  - Перейти в папку `практика 1-6/backend`
  - `npm install`
  - `npm start` (API будет доступно на `http://localhost:3000`)

- `практика 1-6/frontend` — статическая страница `products.html`.
  - Файл: `практика 1-6/frontend/products.html`
  - Откройте в браузере или запустите статический сервер (например `python3 -m http.server 3001`) и откройте `http://localhost:3001/products.html`.

Картинки товаров находятся в `практика 1-6/frontend/assets/images` и ожидаются в формате `.png`.

Если `node_modules` случайно был закоммичен ранее, он удалён из индекса — выполните `git pull` и затем слежуйте инструкциям выше в `backend`.
# Практика 1-6 — репозиторий


В этой папке находятся реализации практических заданий 1–6.

Краткое содержание:
- Практика 1: статическая страница с карточкой товара (SASS). Файл: `index.html`, `styles.scss`, `styles.css`.
- Практика 2: простой API товаров на Express (файл `app.js`).
- Практика 3: тестирование API с помощью скрипта `test_api.sh` (сохраняет ответы в JSON-файлы).
- Практика 4: простой фронтенд, который получает товары с API: `products.html` (сервер статических файлов на порту 3001).
- Практика 5: добавлена документация Swagger по адресу `/api-docs`.

Запуск (локально):

1) Убедитесь, что установлен Node.js (рекомендуется v14+).

2) Установите зависимости (один раз):

```bash
cd "практика 1-6"
npm install
```

3) Запустите API:

```bash
node app.js
```

API доступен: `http://localhost:3000`

Swagger UI: `http://localhost:3000/api-docs`

4) Чтобы протестировать API с командной строки (пример тестов Практики 3):

```bash
./test_api.sh
```

Скрипт сохранит ответы в файлы `test_products_all.json`, `test_product_1.json`, `test_product_created.json`, `test_product_updated.json`.

5) Запустите статический фронтенд (Практика 4) — в отдельном терминале можно запустить простой HTTP-сервер, например:

```bash
cd "практика 1-6"
python3 -m http.server 3001
# затем откройте http://localhost:3001/products.html в браузере
```

Итог: реализованы практики 1–5; Практика 6 — подготовка отчёта и проверка выполнены (см. этот README).
